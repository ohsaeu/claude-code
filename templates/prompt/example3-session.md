# 세션 관리 명령어  

## 현재 세션의 컨텍스트를 간소화 : 대화가 길어져서 응답이 느려질 때
❯ /compact

## 세션 초기화(완전 새로 시작) : 완전히 다른 주제로 넘어갈 때
❯ /clear

## 현재 세션 상태 확인 : 현재 세션에서 사용 중인 메모리나 MCP, 어카운트, 모델 등 상태를 확인할 때
❯ /status

## 현재 컨텍스트 확인 : 현재 세션의 컨텍스트 메모리 할당 현황을 확인하고자 할 때
❯ /context


# 효율성 증대 명령어 : --permission-mode
## 자동 실행 모드

```bash
# 편집 제안을 자동으로 적용(신중하게 사용) - 반복적인 리펙토링 작업, 코드 정리
$ claude --permission-mode acceptEdits "리팩토링을 진행해줘"

# 계획 모드로 먼저 단계 확인 후 실행 - 복잡한 기능 개발, 아키텍쳐 변경, 대규모 마이그레이션
$ claude --permission-mode plan "새로운 결재 시스템을 구현해줘"

# 모든 권한 수락 프롬프트를 무시 - 도구 실행보다 다양한 권한이 반복적으로 요구되는 작업
$ claude --permission-mode bypassPermission "지금 체인지를 변경해줘"
```

# @-파일참조
## 개발 시나리오 예
```bash
# 특정 파일 참조
> @package.json "이 패키지 설정에 맞는 테스트 스크립트를 추가해줘"
> @package.json "프로덕션 빌드를 실행하고 배포 전 체크리스르르 만들어줘"

# 여라 파일 참조
> @src/components/Header.tsx @src/styles/globals.css "Header 컴포넌트의 스타일을 개선해줘"
> @src/components/ "이 컴포넌트를 리뷰하고 개선점을 제안해줘"

# 디렉터리 참조
> @src/utils/ "이 유틸리티 함수들을 TypeScript로 리팩터링해줘"
> @src/api/auth.js "이 api코드에 에러 핸들링을 추가해줘"
```
