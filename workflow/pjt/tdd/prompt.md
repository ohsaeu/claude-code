## 사용자 등록 기능을 TDD로 개발합니다. 먼저 실패하는 테스트부터 작성해줘. 요구사항은 다음과 같아
- 이메일과 비밀번호로 회원가입
- 이메일 중복 체크
- 비밀번호 해싱
- 입력값 유효성 검사

## 작성된 테스트를 통과하는 UserService 클래스를 구현해줘. 최소한의 코드로 테스트만 통과하면 돼.

## UserService 코드를 리팩터링해줘. 의존성 주입, 에러 핸들링 개선, 타입 안정성 강화를 적용하고 테스트도 함께 수정해줘

## 현재 코드의 테스트 커버리지를 분석하고, 누락된 케이스를 추가해줘. 목표 커버리지는 90% 이상입니다.

## Express API 엔드포인트에 대한 통합 테스트를 작성해줘.
- post /api/users - 사용자 생성
- get /api/users - 사용자 조회
- put /api/users - 사용자 수정
- delete /api/users - 사용자 삭제
- supertest를 사용하고, 데이터베이스는 모킹해줘

## 전자상거래 사이트의 구매 플로우에 대한 E2E 테스트를 Playwight로 작성해줘

1. 홈페이지 방문
2. 상품 검색
3. 상품 선택
4. 장바구니 추가
5. 체크아웃
6. 결재 정보 입력
7. 주문 완료 확인

 ---
  실행 방법

  방법 1 — 네트워크가 가능한 환경에서 설치
  cd e2e
  npx playwright install chromium
  npm test

  방법 2 — Docker (가장 확실)
  cd e2e
  docker run --rm --network host \
    -v $(pwd):/e2e -w /e2e \
    mcr.microsoft.com/playwright:v1.58.2-jammy \
    npx playwright test

  방법 3 — Windows 터미널에서 직접 (WSL 파일시스템 접근)
  # PowerShell에서
  cd \\wsl$\Ubuntu\home\oh\temp1\e2e
  npm install
  npx playwright install chromium
  npm test
